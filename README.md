# おがまるパック・おがさわら丸 空き状況監視

2026年9月18日東京発について、次を5分ごとに確認します。

- おがまるパック：いずれかの宿泊施設・部屋に空き表示が出たか
- 船のみ：2等和室または2等寝台に空き表示が出たか

空きが発生した時だけDiscordまたはTelegramへ通知します。満席へ戻った後に再度空きが出れば、もう一度通知します。

## 推奨：Discord通知の準備

1. Discord公式サイトでアカウントを作成し、メール認証を完了します。
2. 左側の「＋」から自分専用サーバーを作ります。
3. サーバー名を押し、`サーバー設定` → `連携サービス` → `ウェブフック` と進みます。
4. `新しいウェブフック` を作り、通知先チャンネルを選択します。
5. `ウェブフックURLをコピー` を押します。このURLは第三者へ公開しないでください。

## GitHubへの設置

1. GitHub公式サイトで無料アカウントを作成し、メール認証を完了します。
2. GitHubで `New repository` を押します。
3. Repository nameを `ogamaru-monitor` にします。
4. 無料運用を優先する場合は `Public` を選択します。Webhook URLはSecretsへ入れるため、公開されません。
5. このZIPを展開し、フォルダー内のファイルをすべてリポジトリへアップロードします。`.github` フォルダーも必要です。
6. リポジトリの `Settings` → `Secrets and variables` → `Actions` → `New repository secret` を開きます。
7. Nameを `DISCORD_WEBHOOK_URL`、SecretにコピーしたDiscord Webhook URLを入力して保存します。

## 通知テスト

1. リポジトリ上部の `Actions` を開きます。
2. 左側の `おがまる空き状況監視` を選択します。
3. `Run workflow` を押します。
4. `空き状況に関係なくテスト通知を送る` にチェックし、もう一度 `Run workflow` を押します。
5. Discordへテスト通知が届けば設定完了です。

その後は5分ごとに自動確認されます。GitHubの混雑状況により開始が遅れる場合があります。

## Telegramを使う場合（任意）

GitHub Secretsへ次の2項目を追加します。

- `TELEGRAM_BOT_TOKEN`
- `TELEGRAM_CHAT_ID`

DiscordとTelegramを両方設定すると、両方へ通知されます。

## 対象日の変更

`.github/workflows/monitor.yml` の次の行を書き換えます。

```yaml
TARGET_DATE: "2026/09/18"
```

## 注意事項

- 公式サイトへの反映にはタイムラグがあります。
- 船の2等客室の空席表示には、レディースルームの空席が含まれる場合があります。
- 空き表示が出ても予約を保証するものではありません。通知後すぐに公式サイトまたは電話で確認してください。
- 自動予約は行いません。
- ページ構造が変更された場合、判定できなくなることがあります。Actionsの各実行に7日間、判定画面の画像が保存されます。
